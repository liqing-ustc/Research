2512-NLAC

Natural Language Actor-Critic: Scalable Off-Policy Learning in Language Space

arXiv preprint·Joey Hong , Kang Liu, Zhan Ling, Jiecao Chen, Sergey Levine

Dec 04, 202511 min read

  • RL
  • web-agent
  • task-planning

Summary

提出 NLAC，用 language-space Bellman backup 训练生成式 critic，实现 LLM agent 的可 scale off-policy actor-critic 学习。

Key Takeaways:

  1. Language Bellman backup: critic 预测”未来 rollout 的文本摘要”，通过 bootstrap 从 (st+1​,at+1​) 的 target summary 获取监督，避免 NLRL 类方法对 on-policy 采样的依赖。
  2. Refinement-based policy improvement: 不对 combinatorial language action 枚举 greedy argmax，而是训一个 refinement policy 读取 critique 生成改进动作，再蒸馏回 π，把 AC update 变成纯监督学习。
  3. 理论桥梁: 在 realizability 假设下，语言 critic 的表示与真实 Qπ 单调同构（Thm 4.5），并且 policy iteration 收敛到最优（Thm A.4），把 successor features 的结构套到 language space。
  4. 实证收益: 相比 PPO/GRPO/NLRL 等 baseline，在 MATH500-Hard、20Q、τ-Bench (retail + airline zero-shot) 上整体最优，7B 上 τ-Retail 从 0.32→0.42，32B 上从 0.49→0.56。

Problem & Motivation

LLM agent 的 multi-turn 任务（web 浏览、tool-use、对话决策）需要长程信用分配，但：

  1. Scalar critic 压缩信息过多：PPO/GRPO 把复杂 language action 的成败压成一个数，没法告诉 policy “哪里错了、怎么改”。
  2. In-context aggregation 不 scale：NLRL 用自然语言 value，但依赖在 prompt 里把多条 on-policy rollout 塞进去做 “language averaging”，受 context length 和采样分布约束，没法 off-policy。
  3. Combinatorial action space：语言 action 空间组合爆炸，传统 AC 的 argmaxa​Q(s,a) 根本无法枚举。

NLAC 要解决的就是：如何在 language space 做真正的 off-policy、可 scale 的 actor-critic 训练。

Method

NLAC 把 critic 从 scalar 换成一段描述未来的自然语言，并用两条 loss 交替训练：policy evaluation（训练 language successor model）和 policy improvement（从 refinement policy 蒸馏）。

核心组件（全部复用同一个 backbone LLM M，只切 prompt）：

  • Policy π: ReAct 风格，输出 thought attht​ + environment action atenv​。
  • Language successor model Mπ: 给定 (st​,at​)，生成未来轨迹 (s,a)t+1:T​ 与终止 reward r(sT​) 的文本摘要 dt​。
  • Language Bellman backup BL​: 从 (st+1​,at+1​,dt+1​) 合成新的目标摘要，相当于在文本空间做 TD backup。
  • Evaluator E: 把 k 条采样的 future summary 聚合成一段 critique QLπ​(st​,at​)。
  • Refinement policy πr: 读初始动作 + critique，产出改进后的 atr​，等价于在 language space 做隐式 argmax。

Figure 1. Overview of NLAC. During policy evaluation, the critic is trained using a language Bellman backup that operates in textual space. During policy improvement, the policy is distilled from a refinement policy.

Equation 1. Language Bellman Backup

Pr[dt​=B(r(st​,at​),st+1​,at+1​,dt+1​)∣st+1​,at+1​,dt+1​]

符号说明：dt​ 是时刻 t 的 language future summary；B 是由 prompt 定义的文本合成算子；st+1​,at+1​∼Pπ(⋅∣st​,at​)；dt+1​∼Mπ(⋅∣st+1​,at+1​)。 含义：把下一步 reward、下一 state/action 和 bootstrapped future summary 组合成当前步的 target 摘要。形式上与 scalar Bellman r+γV(s′) 对应，但操作对象是文本。

Equation 2. Policy Evaluation Loss

L1​(st​,at​,st+1​)=DKL​(BL​Mθˉ​(⋅∣st​,at​)​Mθ​(⋅∣st​,at​))

符号说明：Mθ​ 是当前 successor model 参数；θˉ 是 EMA target 参数，类似 DQN 的 target network；DKL​ 在 token 级别计算。 含义：让 Mθ​ 匹配由 target 网络 bootstrap 出的 Bellman target 分布；EMA 是为了防止生成式 critic 坍塌到 trivial 解。

Equation 3. Policy Improvement Loss

L2​(st​)=−logπθ​(atr​∣st​),at​∼πθ​(⋅∣st​), atr​∼πθr​(⋅∣st​,at​,QLθ​(st​,at​))

符号说明：atr​ 是 refinement policy 在看到 critic 的文本反馈后输出的改进动作；QLθ​ 是 language critic 的 critique 文本。 含义：把 πr 产生的 refined action 当监督信号蒸馏回 π，等价于一次 language-space policy improvement。无需 on-policy rollout，可以直接在 replay buffer 上跑。

Equation 4. 理论桥梁 (Thm 4.5)

Qπ(s,a)=g(QLπ​(s,a))

符号说明：g 是某个单调映射；Qπ 是真实 scalar Q；QLπ​ 是 language critic 的输出。 含义：在 realizability 假设下，学到的 language critic 表示与真实 Q 保持单调一致，因此 refinement-based argmax 不会偏离真实 greedy 方向。这给 “用文本替代 scalar” 提供了理论 backbone。

训练算法 Algorithm 1 交替执行：(1) 环境采样扩充 buffer；(2) 采 batch 同时做一步 L1​ 和 L2​ 的梯度更新。

Implementation Details

  • Base model: Qwen2.5-7B-Instruct 与 QwQ-32B 两档。
  • Policy/critic/refinement 共享权重：同一个 LLM，通过 prompt (p_pred / p_tpred / p_eval / p_refine) 切换角色。
  • EMA target: 防止 successor model 生成式坍塌。
  • 采样数: evaluator 聚合 k 条 future summary；refinement 迭代 m 轮。
  • 对比基线: Prompt-only GPT5-ReAct；fine-tuning 类 RFT、PPO、GRPO、NLRL、Self-Distillation、NLQL。

Insights：

  • NLAC 的优势主要集中在多步、需要信用分配的任务（20Q、τ-Bench）；在单轮 reasoning (MATH500-Hard) 上被 Self-Distillation/PPO 略微反超，说明 language critic 的收益不在”更强的 token-level reasoning”而在”为序列 action 提供可解释反馈”。
  • Zero-shot 到 airline：NLAC (32B) 把 τ-Airline 从 PPO 的 0.41 提到 0.43，比 GRPO/NLRL 都稳。language critique 自带 policy guideline 的 verbal 解释，迁移到 airline policy 时仍然可用。
  • NLRL vs. NLAC：7B 上 τ-Retail 从 0.25→0.42 是巨大跳变，验证了 off-policy bootstrap 相对 in-context aggregation 的 scale 优势。

论文点评

Strengths

  1. Problem framing 到位：指出 “scalar critic + language action” 的 impedance mismatch，并用 “language successor features” 给出一个看起来自然又 grounded 的桥梁。
  2. 真正做到 off-policy：相比 NLRL 必须重新采 rollout 做 in-context averaging，NLAC 用 EMA target + bootstrap 把 replay buffer 用起来，这是走向可 scale 的关键一步。
  3. 理论 + 实验闭环：Thm 4.5 的单调同构与 A.4 的 convergence，虽然依赖较强的 realizability 假设，但至少说明 refinement-based 的伪 argmax 不是 hacky，而是可分析的。
  4. 实验覆盖均衡：同时有 MATH（单轮 reasoning）、20Q（主动信息获取）、τ-Bench（policy-grounded tool use + zero-shot transfer），能把方法的适用面和边界都 probe 出来。

Weaknesses

  1. Refinement policy 的循环依赖：π 和 πr 共享权重，training 时 πr 的输出被当作 π 的监督，若 critique 本身有 bias，会被蒸馏进 policy，形成自我强化。paper 没有 ablate critic 错误率对最终 policy 的影响。
  2. Successor model 崩塌风险处理薄弱：仅用 EMA target 防止 degenerate，但未报告训练后期生成 summary 的 diversity / entropy 曲线。
  3. 计算开销没有公平对比：一次 update 需要 rollout sampling + evaluator 聚合 k 条 + refinement m 轮 + 两条 loss，实际 token-FLOP 远高于 PPO/GRPO，appendix 未给 wall-clock 与 token cost 对比。
  4. MATH500-Hard 上未击败 PPO(32B)/Self-Distil(7B)：paper 解释得较轻描淡写；更合理的叙事应是”NLAC 专攻多步 credit assignment，不以 single-turn reasoning 为目标”，否则表格里加粗的数字会让 reviewer 怀疑 cherry-pick。
  5. 没有开源代码：arXiv v2 未给出 repo 链接，方法本身又依赖大量 prompt 设计（ppred​,ptpred​,peval​,prefine​），复现难度很高。

可复现性评估

  • 代码开源
  • 模型权重开源
  • 训练细节完整
  • 数据集可获取（MATH500、THINGS、τ-Bench 均为公开数据）

整体可复现性偏低：核心 prompts 未附录、无代码、EMA/采样细节零散在正文，复现需要大量反推工作。

关联笔记

对比

  • NLRL: 同样在 language space 做 value learning，但依赖 in-context on-policy aggregation，NLAC 的最大 delta 是 off-policy bootstrap。
  • GRPO: scalar group-relative advantage 的 on-policy 方法，代表主流 RLHF/agent 训练 baseline。
  • PPO: 经典 on-policy scalar AC，NLAC 的直接对照。

方法相关

  • Actor-Critic: 整体框架继承自经典 AC，critic 和 policy 交替更新。
  • ReAct: policy 的 thought + action prompt 格式。
  • Successor Features: Thm 4.5 的理论灵感来源，language critic 类比 SF 的 latent representation。
  • Target Network: EMA target θˉ 沿用 DQN 思想防止生成式坍塌。

速查卡片

Natural Language Actor-Critic (NLAC)

  • 核心: 用 language Bellman backup 训练生成式 critic，把 off-policy actor-critic 搬进 language space。
  • 方法: language successor model 生成未来摘要 + EMA target bootstrap + refinement policy 蒸馏做 improvement，policy/critic/refiner 共享同一 LLM。
  • 结果: 在 20Q、τ-Retail、τ-Airline (zero-shot) 上均优于 PPO/GRPO/NLRL；7B τ-Retail 0.32→0.42，32B τ-Retail 0.49→0.56；sample efficiency 显著高于 PPO。
  • 代码: 未开源。

Notes

  • “Language successor features” 这个类比值得认真追：如果 QL​ 是 SF 的 language counterpart，那能不能像 Universal Value Function Approximator 一样，直接 condition 在 task description 上做 zero-shot？
  • π 和 πr 共享权重是工程取舍。分离两者可能更稳定，但 training cost 翻倍；值得作为未来 ablation。
  • 与 Process Reward Model 的关系：PRM 给 step-level scalar reward，NLAC 的 critic 给 step-level 文本 critique。后者信息量更大但更难蒸馏，比较两者在 long-horizon 任务上的 scaling 曲线会很有意思。
  • 一个可能的”最小可行”复现路径：先在 20Q 上用小 7B，固定 k=m=1，只实现 L1​ 看 language critic 能否学出 sensible critique；再加 L2​。
