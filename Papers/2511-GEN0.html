GEN-0: Embodied Foundation Models That Scale with Physical Interaction

Generalist AI Blog·Generalist AI Team

Nov 04, 202521 min read

  • VLA
  • manipulation
  • cross-embodiment

Summary

Generalist AI 发布 GEN-0——在 270,000 小时真实操作数据上预训练的 embodied 基础模型，报告在 7B+ 规模上观察到 ossification 的相变，并给出下游任务预测误差随预训练数据量的幂律 scaling law。

Key Takeaways:

  1. Ossification 相变出现在 7B 规模：1B 模型在高数据量下出现 ossification（权重无法吸收新信息），6B 开始受益于预训练，7B+ 能内化大规模机器人预训练数据并以几千步 post-training 迁移到下游任务。这个相变比 LLM 领域观察到的 O(10M) 参数临界点大 ~100 倍，作者把它解释为 Moravec’s Paradox 的实证回声。
  2. 预训练数据 → 后训练性能的幂律：对于固定 finetuning 预算，下游验证误差 L(D)=(Dc​/D)αD​ 随预训练数据规模 D 呈幂律下降，在 16 个多任务验证集上一致成立；真机 A/B 评估显示仅用 5.6 小时 task-specific 数据 post-train 时，预训练数据量直接推升成功率，峰值接近 99%。
  3. Cross-embodiment by design：同一个模型已经跑在 6DoF、7DoF 和 16+DoF semi-humanoid（Dexmate VEGA）上，声称无需架构改动。
  4. Harmonic Reasoning 作为 System1/System2 的替代：blog 提出”同步思考和行动”的训练范式，通过异步连续时间的感知/动作 token 交织，避免依赖 System1-System2 架构或 inference-time guidance；但正文没有给出 Harmonic Reasoning 的具体技术细节或消融实验。
  5. 数据规模和基础设施是核心壁垒：270K 小时自有数据集、每周新增 10K 小时、每训练日消化 6.85 年真实经验——blog 大部分技术 claim 都锚定在”数据量级”这个外部不可复核的变量上。

Surpassing the Intelligence Threshold

GEN-0 的 scaling 实验核心结论是 embodied 模型存在一个”smart-enough”门槛：小于某个规模的模型在高数据量下会出现 ossification，即权重随训练进行逐渐失去吸收新信息的能力。Blog 按规模给出三段式的分化：

  • 1B 模型在预训练中早期就饱和，无法吸收复杂多样的 sensorimotor 数据；
  • 6B 模型开始受益于预训练并展现多任务能力；
  • 7B+ 模型能内化大规模机器人预训练，下游只需几千步 post-training 就能迁移。

Figure 1. Scaling GEN-0 model size (different colors) improves performance in terms of next-action validation prediction error (y-axis, lower is better) on a completely-withheld (zero-shot) long-horizon downstream task. 1B 模型清晰呈现早期 ossification，6B 和 7B 则持续改进。x 轴是预训练 compute，归一化到 GEN-0 7B = 1.0。

Blog 声称这是首次在机器人领域观察到 ossification；此前 LLM 文献里这一现象出现在 O(10M) 参数规模，而 GEN-0 的相变点在 O(1B) 以上——作者将此解读为 Moravec’s Paradox 的实证回声：感知和灵巧操作的表达复杂度比抽象推理更高，因此物理智能的激活阈值在 compute 上也更高。

Scaling Laws for Robotics

Figure 1 展示的是预训练期间 model size × compute 对下游 zero-shot 任务的影响。作者进一步问的是另一类 scaling law：预训练收益如何 persist 到 finetuning 阶段。实验设计是取一组在不同预训练数据子集上训练好的 checkpoint，对它们在 16 个 task set 上做语言条件化的多任务 supervised fine-tuning。

Figure 2. 随着预训练数据增加（不同颜色），post-training 期间多任务性能在 validation loss（上图）和 next-action prediction error（下图 4×4 网格，16 个 task set）上一致提升。这些任务包括 dexterity（Lego 搭建）、行业应用（快餐打包）和 generalization（各种 ”_ anything” 任务）。

趋势也迁移到真机：blind A/B 评估显示在仅用 5.6 小时 task-specific 数据 post-train 的低数据域，预训练数据量直接推升成功率；最高峰值出现在”大规模预训练 + 充分 task-specific post-training”同时满足时，某些任务上能到 99%。作者强调为了让对比有效，他们确保预训练和 post-training 数据集人员和环境完全不重叠。

Figure 3. 真机评估下，更多预训练数据（不同颜色代表不同预训练 base）带来更高的下游平均任务成功率（closed-loop policy rollouts，画带 standard error）。左侧是仅用 5.6 小时（1%）task-specific 数据 post-train 的结果，右侧是用完整预训练数据集和全部 550+ 小时 task-specific post-training 数据的最佳模型。

上述趋势可以用幂律拟合。给定固定的 finetuning budget 和下游任务，当预训练数据规模为 D 时，下游验证误差 L(⋅) 可以用形如下式的幂律预测：

L(D)=(Dc​/D)αD​

符号说明：D 是预训练数据规模（以动作轨迹数计）；Dc​ 是 scale 参数；αD​ 是幂律指数。

含义：给定某个下游任务，可以反推”想达到某个预测误差，需要多少预训练数据”或者”多一倍预训练数据能换回多少 task-specific post-training 数据”——作者把它用于和 partner 做预算对齐的决策工具。Blog 举了 Clothes Handling（分拣、整理、扣扣子、挂衣服）这一真实 workplace 任务作为外推示例，声称可以预测 1 billion 动作轨迹下的性能。

Figure 4. Scaling laws 对 post-trained 模型的 next-action prediction error 作为预训练数据规模（以动作轨迹数计）的函数提供良好描述。结合 model size scaling laws，可以用于预测任何下游 post-training 任务的最优预训练 compute/data 分配。

Robotics is No Longer Limited By Data

GEN-0 的预训练语料是 270,000 小时真实操作轨迹，覆盖 1,000+ 家庭、仓库、workplace，由全球遍布的硬件和 1,000+ 数据采集设备/机器人供给，每周新增 10,000 小时且在加速。

Figure 5. GEN-0 训练用的真实世界操作数据比目前已知最大的机器人数据集多若干数量级（as of Nov 2025）。

Mapping the Universe of Manipulation

为了让 GEN-0 能力扩展到”人类能想到的所有操作”，Generalist 在构造他们声称”史上最大最 diverse”的真实操作数据集，横跨家庭、面包房、洗衣房、仓库、工厂等。Blog 展示了一个内部 t-SNE 可视化工具，允许按语言描述在数据集中做近邻检索。

Figure 6. 对 <1% 预训练数据的内部搜索示例。可视化通过 t-SNE 映射语言标签 embedding，给定文本描述后定位近邻区域并随机采样相关视频。

Infrastructure for Internet-Scale Robot Data

为了支撑这一规模，Generalist 自建了硬件、dataloader 和网络基础设施——包括为各数据采集点铺设专用 Internet 线路、多云合同、自研上传机、O(10K) 核规模的多模态数据处理、压缩数十 PB 数据，采用前沿 video foundation model 的 dataloading 技术，使得训练日吞吐达到 6.85 年真实世界操作经验。

Science of Pretraining

从大规模 ablation 中，作者得出两个经验判断：数据质量和 diversity 比纯体量更重要；不同构造的 data mixture 会得到性质差异显著的 checkpoint。Table 1 对比了 8 个 pretraining 数据集（由多个数据外包伙伴采集、按不同 class 组织），在 10 个长时序任务集上做 finetune 后评估 dexterity / applications / generalization 三维表现。

性能通过两个指标度量：动作预测的 validation MSE

MSEval​=∥a⋆−a^∥22​

和 reverse KL divergence（更好地衡量 mode-seeking 行为）。Reverse KL 用 Monte-Carlo 估计：policy 采样 {a^m​}m=1M​ 构成单位方差 Gaussian 混合 q，ground-truth 诱导 p(a)=N(a;a⋆,I)，

DKL​(q∥p)≈M1​m=1∑M​[logq(a^m​)−logp(a^m​)]

符号说明：a⋆ 为 ground-truth 动作；a^m​ 为 policy 第 m 个采样；M 为采样数；q 为 policy 诱导的经验分布，p 为 ground-truth 诱导的分布。

含义：prediction error 衡量点估计的准确度，reverse KL 衡量分布覆盖与 mode-seeking。作者的经验是：低 MSE + 低 reverse KL 的模型更适合 SFT post-training；高 MSE + 低 reverse KL 的模型分布更多模态，对 post-training RL 更友好。

论文点评

Strengths

  1. Ossification phase transition 是一个真正有信息量的实证 claim：如果 Figure 1 的曲线可信，它给出了”embodied model 需要先过 7B 门槛才能真正受益于预训练”的 mental model，这比常见的 “bigger is better” 更可 actionable，也直接影响后续研究社区的资源分配决策。
  2. 把 LLM 领域成熟的 scaling law 范式（预训练数据 → finetuning 收益的幂律）迁移到机器人：Figure 2-4 的实验设计对齐了 LLM 社区的 convention，让结论可比较可延伸。Clothes Handling 幂律外推到 1B 轨迹的预测尝试是一个健康的”risky prediction”动作。
  3. 数据量级是 dominant moat：270K 小时 + 每周 10K 小时的采集速率 + 自建网络/dataloader/多云基础设施——这套 data engine 本身就是独立于模型架构的资产，即便 GEN-0 的模型细节没开源，这个 operation 规模的披露对行业也是有价值的基准线。
  4. Cross-embodiment 作为设计原则而非 afterthought：同一个模型跑 6DoF/7DoF/16+DoF（含 semi-humanoid Dexmate VEGA）的声明，符合 embodied FM 领域的合理期待。

Weaknesses

  1. Harmonic Reasoning 几乎没有技术细节：这是 blog 开篇强调的 “core feature” 之一，正文却没有专门章节、没有架构图、没有 ablation。读者只知道它”涉及 asynchronous continuous-time sensing/acting token 的 harmonic 交互”——不足以判断它是否真能替代 System1-System2 或 inference-time guidance。这是典型的 announcement-level claim，在可复现性上是空白。
  2. 所有关键数字都在 blog 外部不可复核：模型权重、代码、数据集都未公开；partner 被匿名化（A/B/C）；Figure 2-4 的 x 轴都归一化了，没有绝对量纲；Table 1 没有方差/显著性。这意味着”7B 是相变点""MSE 差距对应可预测性能差”这些 claim 都只能 take-as-stated。
  3. Ossification 归因的反事实空白：1B 模型饱和究竟是 capacity 不够、还是优化器超参/学习率 schedule 没为该规模调优？Blog 没有报告是否在 1B 上尝试更长 warmup、更小 learning rate、或 MoE 等替代方案。一个规模的失败可能有多个解释，作者直接选了 capacity 解释。
  4. Moravec’s Paradox 的类比是叙事而非证据：把 “robotics 的相变点比 LLM 大 ~100 倍” 归因于感知和灵巧操作的计算复杂度更高——这是吸引人的故事，但 blog 没给出独立的 complexity-theoretic 或信息论 argument，也没有控制其他可能的混淆变量（如输入 token 密度、data diversity 维度）。
  5. 99% 成功率的 headline 数字披露不完整：blog 说”certain cases”能达到 99%，没说哪类任务、评估 episode 数、是否允许 retry、怎么定义成功。这是典型需要更多细节才能评估的 marketing-flavored statement。

可信评估

Artifact 可获取性

  • 代码：未发布。Blog 没有提及任何 GitHub repo 或 inference demo；Generalist AI 作为商业公司也没有开源公告。
  • 模型权重：未发布。任何规模（1B/6B/7B/10B+）的 checkpoint 均未公开。
  • 训练细节：仅高层描述。正文只说”16 个 task set 做多任务 SFT""5.6 小时 task-specific 数据”等数据量信息，完整超参、学习率、batch size、训练步数、优化器配置、具体架构参数都未披露。
  • 数据集：私有。270K 小时数据为 Generalist 自有；pretraining 数据采集 partner 用 Partner A/B/C 匿名化，外部无法访问或复核。

Claim 可验证性

  • ⚠️ 7B 是 ossification 相变点：基于单张 Figure 1，x 轴 compute 归一化，y 轴是”完全 withheld 的 long-horizon downstream task”的 prediction error——任务身份、评估样本量、是否多 seed 都未说明。归因 ossification 而非 under-tuning 也没有反事实 ablation。
  • ⚠️ 预训练数据 → 下游性能的幂律关系：Figure 2-4 的趋势看起来确实单调，但数据集/模型身份都匿名/归一化，外部无法独立检验幂律指数的稳健性。
  • ⚠️ 某些任务达到 99% 成功率：没有任务身份、评估规模、成功定义；典型的半可信 headline。
  • ⚠️ Harmonic Reasoning 可替代 System1-System2 架构：blog 完全没有给出对比实验或技术细节，这是尚无 grounding 的方法论 claim。
  • ❌ “270K 小时是史上最大机器人操作数据集” / “10K 小时 / 周” / “6.85 年经验 / 训练日”：这些是数据运营声明，外部完全不可独立验证；作为商业基础设施披露可以信，作为技术 claim 需打折扣。
  • ✅ Cross-embodiment 支持 6DoF/7DoF/16+DoF：虽然规模与效果不可核，但 blog 视频 gallery 里有 Dexmate VEGA 的实机视频，作为 demo 级 grounding 成立。

关联工作

基于

  • Scaling Laws for Neural Language Models (Kaplan & McCandlish et al., 2021)：方法论祖本，GEN-0 的幂律拟合直接照搬 LLM scaling law 的形式。
  • Scaling Laws for Transfer (Hernandez et al., 2021)：提供”预训练收益 persist 到 finetuning”的概念框架，也是 GEN-0 幂律声明的参照。
  • Overtrained Language Models Are Harder to Fine-Tune (Springer et al., 2025)：LLM 文献中 ossification 的先例，作者引用它来建立类比但强调自己在 pure pretraining 阶段观察到此现象，而非 pretrain-to-finetune setting。
  • PaLM-E (Driess et al., 2023)：embodied multimodal 的前置范式，Generalist 把 PaLM-E 类 vision-language 预训练定位为 GEN-0 之前机器人 FM 的主流做法。

对比

  • Helix (Figure AI, 2025)：Harmonic Reasoning 明确对比的 System1-System2 架构代表，GEN-0 声称不需要这种双系统拆分。
  • Real-Time Chunking / π0-FAST 类 inference-time guidance (Black et al., 2025)：另一种处理”physics doesn’t stop”的思路，GEN-0 也声称不依赖这类 inference-time 技巧。

方法相关

  • Moravec’s Paradox (Moravec, 1988)：叙事框架，用来解释为什么 embodied 的相变点比 LLM 大几个数量级。
  • Imitation Learning as f-Divergence Minimization (Ke et al., 2020) / Divergence Minimization Perspective on IL (Seyed Ghasemipour et al., 2019)：reverse KL 作为 mode-seeking 度量的理论依据。

GEN-0: Embodied Foundation Models That Scale with Physical Interaction

  • 核心：在 270K 小时真实操作数据上预训练的 embodied FM，报告 7B 规模的 ossification 相变和预训练数据 → 下游性能的幂律。
  • 方法：大规模机器人数据预训练 + 多任务 SFT；Harmonic Reasoning（异步感知/动作 token 交织）；幂律 L(D)=(Dc​/D)αD​ 拟合下游误差。
  • 结果：1B 早期 ossification，6B/7B+ 持续吸收预训练；真机 A/B 显示更多预训练 → 更高下游成功率（峰值 ~99%）；cross-embodiment 在 6DoF/7DoF/16+DoF 机器人上运行。

Notes

  • Blog 开篇 bullet 列表里把 Harmonic Reasoning 列为 core feature，正文却没有对应章节——这是 announcement blog 常见的 “promise more than body delivers” 问题。如果想真正理解 Harmonic Reasoning，只能等后续论文或技术报告。
  • Partner A/B/C 的匿名化让 Science of Pretraining 段退化成”我们试过 8 个数据集，最后这两个 mixture 表现最好”级别的定性结论。更大价值的问题其实是：“什么类型的数据分布改变了什么类型的下游行为”——blog 暗示了答案（Class 1 vs Class 3 的 specific vs diverse 区分）但没量化。
  • Ossification 现象值得和 LLM 领域的 Overtrained LLM (Springer 2025) 做深度对比：规模差异（O(10M) vs O(1B)）到底是”embodied 本质更难” 还是”robotics 的 token 级信息密度不同”？如果后者，Moravec’s Paradox 解释就有点 overclaim。
  • 想追问的问题：GEN-0 的模型架构到底是 unified transformer + action head？还是 VLA 风格的 PaliGemma 类 backbone + flow matching head？Blog 完全没说。这影响对 “10B+ embodied FM” 这个数字的解读。
