π0.7: a Steerable Model with Emergent Capabilities

  • 核心: 用 diverse multimodal prompts（语言、metadata、控制模态标签、视觉 subgoal）统一异构数据训练出的 steerable 通用 VLA，首次展现 LLM 风格的 compositional generalization
  • 方法: 在 prompt 中加入 “how to do it” 的多维 conditioning，使得劣质自动采集数据、跨机器人数据、人类视频都能被纳入统一训练；推理时用 world model 产生视觉 subgoal、high-level policy 产生语言 subtask
  • 结果: 单一模型达到 π*0.6 专家策略的成功率和 throughput；在无 UR5e 叠衣数据的情况下跨本体完成叠衣，成功率匹配有 375h 经验的 teleoperator 的 “zero shot”；对未见过的气炸锅等厨房电器表现出组合泛化
  • Sources: website

Summary

π0.7 通过 diverse multimodal prompt 统一异构数据训练，在通用模型下同时获得专家级 dexterity、跨本体迁移和 LLM 式 compositional generalization 的早期迹象。

Key Takeaways:

  1. Diverse conditioning 是利用异构数据的钥匙：prompt 不再只说 “做什么”，还说 “怎么做”（speed、quality、control modality、subgoal image），这样连次优的自主采集数据都能被安全纳入训练
  2. Compositional generalization 初现：模型从未见过 “把红薯放进气炸锅” 的示范，但通过语言 coaching 可以完成；最接近的训练数据只是两段关气炸锅的 episode 和 DROID 上的 Franka 数据
  3. Cross-embodiment 突破：静态双臂采集的叠衣数据迁移到 bimanual UR5e（尺寸/形态差异巨大），成功率与 375h 经验的 teleoperator 初次上手新机器时相当
  4. Generalist 取代 Specialist：不再需要像 π*0.6 那样对每个任务 RL 微调——把 Recap 产出的自主数据通过 strategy metadata 蒸馏进通用模型即可达到同等或更高 throughput
  5. Hierarchical inference：高层 policy 生成语言 subtask，world model 合成 subgoal 图像，低层 π0.7 执行动作——把语言 coaching 转为自主执行而无需新 teleop

Teaser. π0.7 官方宣传片，展示模型在多种机器人 / 场景下的通用 dexterity。

Leveraging more data with diverse conditioning

π0.7 将异构数据源（不同机器人、不同控制模态、人类视频、自主 rollout 数据）统一在单一 prompting 框架下，prompt 可包含：

  • Diverse language：任务描述和 sub-step 语言
  • Metadata：描述任务如何被完成，例如 speed、quality
  • Control modality labels：指示使用 joint control 还是 end-effector control
  • Visual subgoal images：当前 sub-step 结束时场景应该长什么样——测试时由轻量 world model 合成

这让原本会教坏模型的次优自主 evaluation 数据可以通过 “低质量 / 低速” 的 metadata 标注被纳入训练而不污染行为分布。推理时：高层 policy 产生语言 subtask，world model 产生 subgoal 图像，desired metadata 指定期望的 speed/quality，π0.7 action expert 生成动作。

Compositional task generalization

最具说服力的场景是操作未见过的厨房电器。让机器人操作气炸锅（air fryer）时，训练集中没有该任务的示范：

  • Zero-shot prompt（“load a sweet potato into the air fryer”）：尝试但失败
  • Step-by-step language coaching：按步骤给出语言指令（类似第一次使用电器的人类会接收的指导），模型成功完成
  • Fine-tune high-level policy：用多次 coaching 数据微调高层 policy 后，它可自主产生语言 subtask，π0.7 完全自主执行——无需任何新 teleoperation

作者反向追溯了训练数据里最接近的 episode：只找到两段在家里关气炸锅的片段（“push the frying basket into the airfryer”）和 π0.5 使用的开源 DROID（Franka）数据。物理动作与 mobile robot 实际执行的动作相差甚远——这佐证了模型是在组合行为，而非模式匹配。

作者把这类比为 LLM 的 compose：知道 English→French 翻译 + 知道产 JSON，就能产出 JSON 格式的翻译。

Cross-embodiment transfer

在训练集中代表性最差的 bimanual UR5e（两台工业臂 + Robotiq 夹爪，惯性大、夹爪精度低，teleop 也很难）上测试叠衣任务。训练数据完全没有 UR5e 叠衣的示范，叠衣数据来自一台小得多的静态双臂机器人。

Video 1. Cross-Embodiment Transfer —— π0.7 驱动 bimanual UR5e 完成训练集中从未见过该本体的叠衣任务。

由于两台机器人在尺寸、定位和形态上差异显著，π0.7 必须采用实质不同的策略。作者给出的 reference 是：让那些在原机器人上采集了训练数据、平均 375 小时 teleop 经验的专家，第一次使用 UR5e 做同样任务，π0.7 的成功率与这些专家的 “zero-shot” 成功率相当。

这是一个相当 striking 的 framing——把模型的跨本体迁移能力锚定到人类专家迁移的水平。

Conditioning on speed and optimality

与 Recap（见 π*0.6）的对比：Recap 是通过 RL 优化单任务 specialist 的 robustness 和 throughput 的算法。π0.7 把 Recap 训练中产生的 experience 通过 strategy metadata distill 进单一通用模型，在叠衣（T-Shirts and Shorts / Diverse - Hardest Item）、意式咖啡、装箱四个任务上同时达到与 specialist 相当或更高的 success rate 和 throughput。

Video 2. Specialist-Level Dexterity —— 单一通用 π0.7 制作意式咖啡，throughput 与 RL 微调的 specialist 相当或更高。

即：通用模型不再输给 RL 微调的 specialist。定量图展示的是 normalized throughput（用 specialist 归一化）和 success rate(%)——具体数字博客未给出精确表格。

Out-of-the-box performance on dexterous tasks

π0.7 可以控制多种机器人执行大量多样化任务（削菜、用 Windex 擦玻璃门等），博客展示了一个选段，但未给出成功率数字。

Video 3. Out-of-the-Box Performance Without Fine-Tuning —— π0.7 零微调地完成切西葫芦。

What’s next?

作者展望：可 steerable 的通用模型 + 丰富的 prompt 形式可能让未来的系统 “think through” 完成未见过的复杂任务——高层推理 → 基于 prompt 把推理 ground 成动作 → 观察结果再修改计划。prompt following 的意义不只是让用户更精确地表达意图，更是把 foundation model 的 semantic generalization 翻译成 physical generalization 的通道。

论文点评

Strengths

  1. 问题 framing 精准：抓住了 VLA 领域一个真正的 gap——semantic generalization 强但 compositional generalization 弱。论文没有只追 benchmark 数字，而是直指 “LLM 式的概念组合” 这个质变指标。
  2. Diverse conditioning 的 insight 简洁且通用：“把 how 也写进 prompt” 解决了异构数据混用时策略冲突的根本矛盾。这个思路对几乎所有 imitation-learning from heterogeneous data 的工作都适用。
  3. Cross-embodiment 的对比 anchor 有信息量：把模型成功率 anchor 到 “375h 经验专家初次上手新机器人” 的 zero-shot 表现，比单纯报数字更能让人理解迁移的难度。
  4. Specialist → Generalist 的收编：通过 metadata conditioning 把 Recap 的 RL 收益 distill 进通用模型，意味着 VLA 的迭代路径不需要每个任务维护一个 specialist checkpoint。

Weaknesses

  1. 无 quantitative 细节：作为 blog 完全缺少 success rate / throughput 的具体数值表、实验规模（trial 数、task 数）、对照组设置。四个任务的 bar chart 只标了 y 轴范围。
  2. Compositional generalization 证据偏 anecdotal：气炸锅案例是 compelling 的 narrative，但作者自己承认 “size and diversity of training set makes it hard to track down the precise episodes”。无法系统评估组合泛化的覆盖范围和失败模式。
  3. Emergent 的定义模糊：博客反复强调 “first signs of compositional generalization”，但没有操作化的衡量标准。VLM / LLM 的 in-context composition 在 π0.5 也有展示过，差异究竟在哪需要正式 paper 才能判断。
  4. World model 细节缺失：推理 pipeline 里 world model 产生 subgoal 图像是关键，但完全未披露架构、训练方式、生成质量评估。
  5. 不开源：blog 形式，无 arxiv 无 github 无 checkpoint，完全无法复现。openpi 仓库当前没有 0.7 版本。

可信评估

Artifact 可获取性

  • 代码: 未开源（openpi 目前只到 π0.5）
  • 模型权重: 未发布
  • 训练细节: 仅高层描述——提到异构数据来源（多机器人、人类视频、自主 rollout）和 conditioning 维度，但无数据配比、训练步数、超参
  • 数据集: 部分公开（DROID 是唯一明确提到的公开子集；其余为内部采集，包括 mobile robot、静态双臂、UR5e 等）

Claim 可验证性

  • ⚠️ “单模型匹配 Recap specialist 的成功率和 throughput”：bar chart 存在但未披露具体数字和 trial 数，无法独立核实
  • ⚠️ “跨 UR5e 叠衣匹配 375h teleop 专家 zero-shot”：对照组设计合理，但无样本量、置信区间，“matches” 的定义模糊
  • ⚠️ “compositional generalization 的早期迹象”：气炸锅案例定性 compelling，但作者承认无法追溯完整来源数据；没有设计专门的组合泛化 benchmark
  • ❌ “step-change in generalization” / “first signs of compositional generalization”：marketing 口吻，缺少 vs 前作（π0.5、π*0.6）的系统对照

关联工作

基于

  • π0：初代 VLA flow 模型，π0.7 的架构祖先
  • π0.5：引入异构 co-training 和 web 数据，π0.7 的直接前身
  • π*0.6：Recap (RL with Experience and Corrections via Advantage-conditioned Policies)，π0.7 通过 metadata 把其 experience distill 进通用模型
  • Hi Robot：高层 policy 生成语言 subtask 的思路，被 π0.7 在推理 pipeline 里沿用

方法相关

  • DROID: 开源 Franka 数据集，π0.7 训练数据之一
  • Subgoal conditioning: 让 world model 生成图像 subgoal 作为 VLA 的额外 prompt，是本工作把 LLM-style prompting 搬进 embodied control 的核心抽象
  • Strategy metadata distillation: 把 RL-trained specialist 的 rollout 加上 “high quality / high speed” 标注反哺通用模型，避免次优数据污染

Notes

  • 最关键的 takeaway 是 prompting 范式：如果 “how” 可以被 prompt 表达，训练数据的质量就不是硬约束而是可控 condition。这把 imitation learning 的 “garbage in, garbage out” 重新框定为 “label the garbage properly”。
  • World model 产生 visual subgoal 这件事值得单独深究：它事实上让 VLA 获得了类似 LLM 的 “chain-of-thought in pixel space” 能力。这条线可能是下一代 VLA 的关键分叉点。
  • Compositional generalization 的系统评估是 open problem。VLA 社区需要类似 SCAN / COGS 那种专门测试组合的 benchmark，否则 “emergent composition” 永远只能靠 anecdote。
