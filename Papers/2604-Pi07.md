## Summary

> [!summary] π0.7: a Steerable Generalist Robotic Foundation Model with Emergent Capabilities
> - **核心**: 用 *diverse context conditioning*（subtask 文本 + strategy metadata + 生成式 subgoal image + control-mode 标签）把异构数据的 "how" 写进 prompt，让单一通用 VLA 同时获得专家级 dexterity、跨本体迁移和 compositional generalization
> - **方法**: 5B VLA = Gemma3 4B VLM backbone + MEM-style video history encoder + 860M flow-matching action expert；训练时 prompt 各组件独立 dropout，推理时可对 metadata 应用 classifier-free guidance；subgoal 由 14B BAGEL 初始化的轻量 world model 异步生成
> - **结果**: 单模型在四个 Recap 任务上 throughput 追平或超过 RL-specialist；UR5e 叠衣零数据迁移 task progress 85.6%（人类 375h 专家 90.9%）；气炸锅 / 烤面包等全新长程任务可通过语言 coaching + high-level policy 微调实现完全自主执行；"no metadata" 与 "no eval data" 消融在 throughput 上有明显 gap
> - **Sources**: [paper](https://www.pi.website/download/pi07.pdf) | [website](https://www.pi.website/blog/pi07)

**Key Takeaways:**
1. **Diverse context conditioning 是利用异构数据的钥匙**：subtask 语言 + strategy metadata（speed / quality / mistake）+ subgoal image + control-mode 标签把策略冲突的数据去歧义化。无 conditioning 时 "更多更差的数据" 反而掉点（Fig. 18 left）
2. **Subgoal image 作为 "pixel-space CoT"**：BAGEL 初始化的 world model 把 web-scale 视觉先验翻译成 robot-task subgoal，尤其在破 dataset bias（Reverse Fridge→Microwave）和跨本体叠衣任务上 critical
3. **Specialist → Generalist 的反向 distill**：把 [[2511-PiStar06|π*0.6]] RL 训练产生的 rollout 标 "high quality / high speed" 再喂回通用模型，通用模型 throughput 甚至超过 RL specialist（diverse laundry / box building）
4. **Cross-embodiment = 策略涌现，不是运动 copy**：UR5e 叠衣采用与源 robot 不同的垂直夹持策略，task progress 匹配 375h 经验 teleoperator 首次上手的 zero-shot baseline
5. **Language coaching → 新自主能力**：无需新 teleop，coaching 数据即可微调 high-level policy 让气炸锅、烤面包等长程任务完全自主，autonomous ≈ coaching 表现

**Teaser. π0.7 概览视频** —— 单模型驱动多种本体完成精细操作到长程家务。

## Introduction & Motivation

作者把 VLA 的核心 gap 诊断为：LLM 风格的 compositional generalization 在机器人基础模型上长期缺席。Prior VLA 往往连训练分布内的全部指令都需要 task-specific fine-tuning 才能流利执行，更遑论将技能重新组合解新任务。

π0.7 的假设：**不是加更多数据，而是把数据的 context 写清楚**。每段 episode 除了 "做什么"，还标注 "用了什么策略、质量如何、速度多少、是否出错、用了什么控制模式、目标长什么样"。这样训练的模型可以：

- 学习次优数据（失败、自动 rollout、跨质量 demo）而不被污染
- 吸收非机器人数据（egocentric 人类视频、web 多模态数据）
- 测试时被精确 "steer" 到想要的策略
- 以 "新方式组合旧技能" 处理新任务

论文总结四类 emergent 能力：Out-of-the-box performance、Instruction generalization、Cross-embodiment generalization、Compositional task generalization。

## Flow-based VLA 背景

π0.7 建立在 [[2410-Pi0|π0]] / [[2504-Pi05|π0.5]] / π0.6 的 flow-matching VLA 传统上：VLM backbone + transformer action expert，action expert 用 flow matching 预测 action chunk $a_{t:t+H}$。训练采用 knowledge insulation (KI) recipe——VLM backbone 被 FAST tokens 监督，action expert 可 attend VLM 激活但梯度不回流到 backbone，这样 VLM 以稳定的交叉熵损失训练，同时把表征借给 action expert。

**Equation 1. 训练目标**

$$
\max_\theta \mathbb{E}_\mathcal{D} \left[\log \pi_\theta (a_{t:t+H} \mid o_{t-T:t}, C_t)\right]
$$

**符号说明**：$a_{t:t+H}$ 为 action chunk；$o_{t-T:t}$ 为 T 步历史观测；$C_t$ 是 prompt/context，传统上只含语言指令 $\ell_t$，π0.7 扩展为 multimodal prompt。
**含义**：flow matching 实际优化下界而非精确 log-likelihood。

## Diversifying the Prompt

π0.7 的核心方法贡献：把 context $C_t$ 从单一 task description 扩展为以下多模态组件，**每个组件在训练时独立 dropout**，推理时灵活组合：

### Subtask instructions（延续自 [[2504-Pi05|π0.5]]）

在整体 task $\ell_t$（如 "clean the kitchen"）之外，加入当前 sub-step $\hat\ell_t$（如 "open the fridge door"）。推理时 $\hat\ell_t$ 可由学到的 [[2502-HiRobot|Hi Robot]] 式 high-level policy 产生，或由人类 live coaching。作者声称正是这个设计让 **语言 coaching** 成为可用的教学接口。

### Subgoal images

多视角 subgoal $g_t = [G^1_t, \dots, G^n_t]$ 描述 near-future 期望场景。Wrist view 捕捉 gripper 状态，base view 捕捉物体布局。推理时由独立的 world model $p_\psi$ 生成——initialized from **BAGEL (14B mixture-of-transformers)**，沿用 SuSIE 风格继承 web-scale image editing 先验。

**Equation 2. World model 训练目标**

$$
\max_\psi \mathbb{E}_{\mathcal{D}_g} \left[\mathcal{L}_{\mathrm{CFM}}\left(g_t^\star,\, g_\psi(o_t, \hat\ell_t, m)\right)\right]
$$

**符号说明**：$g_t^\star$ 是真实 future subgoal 图；$\hat\ell_t$ 是 subtask 文本；$m$ 是 episode metadata；$\mathcal{L}_{\mathrm{CFM}}$ 是 flow matching loss；$\mathcal{D}_g$ 是带高质量 subtask label 的数据子集。
**含义**：world model 用图像生成模型初始化，继承 web-scale 视觉先验后在 robot data 上微调，生成 subgoal 以 disambiguate 语言 prompt。

### Episode metadata

给每段 trajectory 打三类离散标签：
- **Overall speed**: episode 长度（timesteps），500 步为一档
- **Overall quality**: 1-5 分人工评分
- **Mistake**: 布尔，是否出错

这让失败 episode / 次优自主 rollout 可被 "标注" 而非 "丢弃" 后纳入训练。测试时 metadata 可用于 steer：**对 metadata 应用 classifier-free guidance (CFG)** 进一步放大对期望策略的偏置。

**Equation 3. Metadata 上的 CFG 推理**

$$
\nabla_a \log \pi_\theta(a_{t:t+H} \mid o_t, C_t) + \beta \big( \nabla_a \log \pi_\theta(a_{t:t+H} \mid o_t, C_t) - \nabla_a \log \pi_\theta(a_{t:t+H} \mid o_t, C_t^{\text{uncond}}) \big)
$$

**符号说明**：$C_t^{\text{uncond}}$ 是去掉 metadata 部分的 unconditional context；$\beta \in \{1.3, 1.7, 2.2\}$ 是 CFG 权重。
**含义**：CFG 在去噪过程中放大 conditional 与 unconditional 的差异，把生成动作向 high-quality / high-speed 模式偏置。任意 prompt 组件理论上都可应用，作者主要在 metadata 维度上使用。

### Control modality label

$c \in \{\text{joint}, \text{ee}\}$，指示 joint 还是 end-effector 控制。不同任务选择不同模式。作者在附录注明 prior π0.5 / π0.6 没能从 ee 模式中获益，而 π0.7 可以（因为 context 足够 disambiguate 两种模式）。

### 完整 prompt 与 dropout 策略

```
<Multi-view observation><Multi-view subgoals>
Task: peel vegetables. Subtask: pick up the peeler.
Speed: 8000. Quality: 5. Mistake: false. Control Mode: joint.
<Proprioception>
```

训练时 dropout 比例：subgoal 图只给 25% batch（否则 "inverse dynamics" 捷径让其他 context 学不到）；有 subgoal 时 30% 概率 drop subtask text；metadata 整体 drop 15%，每个子组件独立 drop 5%；control mode 不 dropout。

## Architecture（Fig. 2 总览）

π0.7 总参数 ~5B：
- **VLM backbone**: Gemma3-4B（含 400M vision encoder），block-causal masking（observation / subgoal tokens 双向，文本 causal）
- **History encoder**: 继承 [[2603-MEM|MEM]] 的 video history encoder，最多 4 个相机 × 6 帧历史，时空压缩到单帧 token 数；history 整体以 0.3 概率 drop
- **Subgoal 编码**: 同一 vision encoder，448×448 分辨率，subgoal token 可 attend observation
- **Proprioception**: 线性投影 joint state 到 backbone 维度（不同于 π0.6 用 discrete tokens）
- **Action expert**: 860M transformer，flow matching，50 token 对应 50 步 action chunk；用 **adaptive RMSNorm** 注入 flow timestep
- **训练时 RTC**: 模拟 0-12 timesteps 的推理延迟（对应 50 Hz 机器人 240 ms 延迟）

### Runtime algorithm

**Algorithm 1（简化）**：每步异步刷新两条流水线——subgoal image 由 world model 按 "subtask 变化" 或 "Δ=4 s 超时" 触发重生成；VLA 每 $\hat H$ 步出一个 action chunk；action inference 与 subgoal / subtask 生成并行。推理常用 5 denoising steps，执行 $\hat H \in \{15, 25\}$ 步。

> ❓ 作者没给出 world model 推理延迟 vs action expert 延迟的量化 breakdown。附录里给了 minimal 变体 38 ms / 完整 variant 127 ms，但 14B BAGEL 单张 subgoal 1.25 s（25 denoising steps，4×H100 张量并行 + 8-bit 量化 + SageAttention）才是真正成本——4 s 异步刷新算下来 world model 占据 ~31% 的可用时间窗口，对实时部署是非常窄的 budget。

## Training Data

覆盖机器人 demo（静态 / 移动，单臂 / 双臂）、在野家庭环境、大量策略自主评估数据（含 RL 训练的 [[2511-PiStar06|π*0.6]] rollout）、人类干预、开源机器人数据集（DROID）、egocentric 人类视频、web 多模态数据（VQA、object localization、attribute prediction、视频描述）。

论文点明与经典 VLA 训练管线的最大区别：**大量使用次优机器人数据**——失败 episode、有错误的成功 episode、prior model 的 rollout。这相当于把 specialist 的 "经验" distill 进 generalist。

> ❓ 数据配比、总小时数、训练步数仍未公开。

## Experimental Evaluation

实验围绕四条主轴 + 一组消融：out-of-the-box dexterity、instruction following、cross-embodiment、compositional task generalization、data / context scaling。

### A. Out-of-the-box dexterous performance（Fig. 6）

在 [[2511-PiStar06|π*0.6]] 发布时评测的四个任务（T-Shirt / Shorts 叠衣、Diverse Hardest Laundry、意式咖啡、Box Building）上，单一 π0.7 模型直接追平或超过 task-specific RL specialist 的 success rate 和 normalized throughput。在更多 "Robot Olympics" 任务（花生酱三明治、反穿 T 恤、开门、切黄瓜、削菜、扔垃圾）上匹配 SFT specialist。

**Video 1. Specialist-Level Dexterity** —— 单一通用 π0.7 制作意式咖啡，throughput 与 [[2511-PiStar06|π*0.6]] RL 微调 specialist 相当或更高。

### 消融：metadata 与自主评估数据的贡献（Fig. 7）

四个 [[2511-PiStar06|π*0.6]] 任务上对比：
- **π0.7**（完整）
- **π0.7 (no metadata)**：移除 episode metadata
- **π0.7 (no eval data)**：训练时排除所有自主 evaluation episode

π0.7 全面胜出，throughput 上 gap 最显著。结论：**自主 rollout 必须 + metadata 必须**——两者结合才能把混质数据转化为增益；移除任一项，混入次优数据反而伤模型（与 Fig. 18 left 呼应）。

### 记忆任务（Fig. 8）

在需要显式记忆的任务（Swap 3 Mugs、Find Object、Scoop Coffee、Window Cleaning）上对比 [[2603-MEM|MEM]] paper 中 π0.6-MEM 的 SFT specialist。π0.7 未做 task-specific finetune 即匹配或超过 specialist——**MEM-style history encoder 已内化**。

### B. Instruction following（Fig. 9, 10, 11）

#### 新环境下的广泛指令跟随（Fig. 9）
4 个 unseen 厨房 + 2 个 unseen 卧室，14 个 3-6 步指令场景。π0.7 显著超过 [[2504-Pi05|π0.5]] 和 π0.6。

#### 复杂 referential 指令（Fig. 10）
"Office Desk Object Rearrangement" 任务分标准 / 复杂两组。标准指令（"pick up the grapes"）所有模型都 ok；复杂指令（"pick up the object I would use to eat soup"、"pick up the fruit on the largest plate"）π0.7 显著更强，**加入 subgoal image（π0.7 (GC)）再涨一截**——world model 把 web-scale 视觉理解带了进来。

#### 打破数据集偏见（Fig. 11）
两个精心构造的 reversed task：
- **Reverse Bussing**: 把垃圾放 bussing bin、餐具放垃圾桶（与训练数据相反）
- **Reverse Fridge to Microwave**: 把食物从微波炉放冰箱（数据集里从未出现反向）

π0.7 显著超过 π0.5 / π0.6。Reverse Fridge→Microwave 上 **subgoal image 几乎是成功的 must-have**——world model 能基于文本生成正确布局的图像，用图像 override 语言层面的数据偏置。

> ✏️ 这是一个优雅的 "visual chain-of-thought" 证据：当语言指令与训练 policy 的默认行为冲突时，图像目标可作为外部 anchor 把 policy 拉回。

### C. Cross-embodiment transfer（Fig. 12, 13）

分两档难度：
- **Object rearrangement**（Table Setting、Bag in Backpack、Organize Tupperware、Shirt Bagging）：随 embodiment gap 增大，π0.5 先崩，π0.6 顶到中等 gap，π0.7 在最大 gap（静态双臂 → UR5e 单臂）上最强
- **Dexterous folding**（UR5e 叠 t-shirt / 毛巾）：训练数据全部来自小静态双臂，**零 UR5e 叠衣数据**。π0.7 能迁移，加上 subgoal image（π0.7 (GC)）进一步提升

关键 framing——**用 375 h 经验的 teleoperator 首次上手 UR5e 的成功率作为对照**：
- 人类专家 zero-shot：task progress 90.9%，success rate 80.6%
- π0.7 (GC) zero-shot：task progress 85.6%，success rate 80%

**Video 3. Cross-Embodiment Transfer** —— π0.7 驱动 bimanual UR5e 完成训练集中从未见过该本体的叠衣任务，匹配 375 h 经验 teleoperator 的 zero-shot baseline。

<video src="https://website.pi-asset.com/pi07/shirt_folding_processed_no_overlay_compressed.mp4" controls muted playsinline width="720"></video>

更有趣的是 Fig. 13 指出的 **emergent strategy**：
- **Shirt bagging**: 静态双臂 teleop 用一臂撑袋、一臂插入；π0.7 在 UR5e 上改用单臂 pick-and-place（利用 UR5e 更长的可达域）
- **T-shirt folding**: 源机器人 teleoperator 用倾斜夹持；π0.7 在 UR5e 上改用垂直夹持（适配 UR5e 臂放置）

模型不是在复制轨迹，而是根据本体学出新策略。

### D. Compositional task generalization（Fig. 14-17）

#### 短程新任务直接零样本（Fig. 17）
按压 French press 活塞、把米舀进电饭煲、擦办公用品、旋转齿轮 / 桌扇——**这些任务都没对应训练数据**，π0.7 直接工作；加 subgoal image 持平或小幅提升。

#### 长程新任务通过语言 coaching（Fig. 14-16）
三个多阶段厨房任务（Loading / Unloading Air Fryer、Toasting a Bagel）：
- **Step 1**: zero-shot prompt "load a sweet potato into the air fryer" 失败
- **Step 2**: 人类 step-by-step 语言 coaching → π0.7 能完成；prior models 在 coaching 下仍失败（语言跟随能力不够）
- **Step 3**: 用 coaching data finetune high-level policy → 后续完全自主执行，与 coaching 表现相当（Fig. 16）——**零额外 teleop**

### E. Scaling ablation: 数据多样性 ↔ 性能（Fig. 18）

#### 左图：混质数据 ablation（Laundry T-Shirts/Shorts）
按 quality + speed 评分分 4 档：top 30%、top 50%、top 80%、all。训练 8 个模型（× 有无 metadata）。结论：
- **π0.7 (no metadata)**: 数据量↑ 但平均质量↓，性能 **下降**
- **π0.7 (with metadata)**: 同样的数据，性能 **持续上升**

即 metadata 让 "更多更差的数据" 也能被吸收——这是 diverse conditioning 最有力的正面证据。

#### 右图：任务多样性 ablation
- **π0.7 (w/o most diverse 20%)**: 移除最高 task diversity 的 20% 数据
- **π0.7 (w/o random 20%)**: 对照，随机移除 20%

在短程未见任务（Fig. 17 那组）上评估：π0.7 和随机移除 20% 接近，but w/o most diverse 20% 显著掉点。说明 **通用化的提升确实来自任务多样性**，不是数据量本身。

## Discussion

作者的核心 takeaway：**diverse prompting + 更大更 diverse 数据 = 把多质量 policy 一起压进一个 pre-trained 模型**，可以反向 distill specialist，并解锁 cross-robot / compositional generalization。

局限和未来：
- Zero-shot 未见任务成功率 60-80%，远低于 in-distribution 的 90%+
- 作者明确建议 "高 steerability + 少量 test-task 数据 / RL" 作为下一步
- Compositional vs remixing 的哲学模糊，作者坦承并反守为攻："组合就是泛化，不必硬区分"

## 论文点评

### Strengths

1. **Problem formulation 精准**：抓住了 VLA 最 fundamental 的 gap——semantic 理解强但 compositional action generation 弱。没有去追 +N% benchmark，而是直指 "LLM 式组合" 这个质变指标。
2. **Diverse conditioning 的 insight 简洁且可迁移**："把 how 写进 prompt" 是个 elegant 的统一视角：demo / 失败 / 自主 rollout / 人类视频 / 跨本体数据共享一个 prompt 空间，用 metadata 和 subgoal 去歧义化。适用于几乎所有 imitation-learning from heterogeneous data 的工作。
3. **Ablation 扎实**：no-metadata / no-eval-data / no-most-diverse-20% / no-random-20% 是直接回应 "diverse conditioning 是否必要"、"混质数据是否必要"、"任务多样性是否必要" 三个关键消融。尤其 Fig. 18 left——**"更多更差数据在无 metadata 下掉点、在有 metadata 下涨点"** 是真正的 causal 证据。
4. **跨本体的人类对照 anchor 有信息量**：375 h 专家首次上手 UR5e 的 zero-shot 作为 baseline，比单纯报 success rate 更能让人把握迁移难度。
5. **Language coaching → autonomous 的闭环**：coaching data 微调 high-level policy 从而让任务完全自主，**完全避开 teleop 成本**。这是一条可规模化的教学管线，比传统 "新任务 = 新采集" 的范式提效一个数量级。
6. **World model 作为 "pixel-space CoT"**：Reverse Fridge→Microwave 必须靠 subgoal 才能破 data bias、UR5e 叠衣靠 subgoal 进一步涨分——把 web-scale 视觉先验作为 prompt 的一部分而非单纯骨架，实操上非常干净。

### Weaknesses

1. **Compositional vs remixing 的辩护避重就轻**：作者承认无法干净区分 "真新任务" 与 "训练数据中远缘技能的组合"，然后转而主张 "不必区分"。这在工程意义上合理，但把 "first signs of compositional generalization" 这个定性 claim 的 epistemic 基础削弱了一大半。
2. **缺少与同类 diverse conditioning 工作的系统对比**：SuSIE、[[2502-HiRobot|Hi Robot]]、CogAct 等都用过某种 subgoal / subtask conditioning；π0.7 在 "多维 metadata + CFG on metadata" 维度上新颖，但没有针对单一 conditioning 维度的对比消融，不易判断哪个维度贡献最大。
3. **数据量、训练成本、world model 成本不透明**：训练 token 数、GPU 小时、world model 推理延迟等部署关键量大部分缺失（仅给出 minimal 38 ms / full 127 ms 的 VLA 推理时间和单张 subgoal 1.25 s）。对完整复现性和 resource 评估构成硬障碍。
4. **BAGEL dependency 存疑**：14B MoT 世界模型是 π0.7 的核心推理组件，但缺少 ablation——"把 BAGEL 换成更小 / 更弱 image 生成器" 性能会怎样？这关系到 subgoal pipeline 是否可被更小模型替代。
5. **Metadata 标注的人工成本**：quality 1-5、mistake 布尔这些都是人标，数据规模上升时标注成本线性上升，论文未讨论自动化路径。
6. **Zero-shot 成功率 60-80%**：虽然相对 prior 有巨大提升，绝对数字对 production-grade 部署仍远远不够。

### 可信评估

#### Artifact 可获取性

- **代码**: 未开源（openpi 当前公开到 [[2504-Pi05|π0.5]]，未包含 π0.7）
- **模型权重**: 未发布；π0.6 model card 已发，π0.7 未见
- **训练细节**: 仅高层描述——架构参数清楚（5B / 4B / 860M / 14B BAGEL / 50 token chunk / 5 denoising steps），但 token 数、训练步数、数据配比、GPU 资源完全未披露
- **数据集**: 部分公开（DROID 是唯一明确 named 的开源子集；大量内部 mobile / 静态双臂 / UR5e 数据 + egocentric video + web data 为私有）

#### Claim 可验证性

- ✅ **"在 Recap 四任务上追平或超过 RL specialist throughput"**：Fig. 6、Fig. 7 给出 bar chart 和 ablation，虽未列精确数字，但 trend 一致且消融充分，可信度高
- ✅ **"Metadata 必要 + 自主数据必要"**：Fig. 7 与 Fig. 18 left 的消融 + scaling 对比是 causal 证据，claim 得到直接验证
- ⚠️ **"匹配 375 h 专家 zero-shot on UR5e 叠衣"**：10 人 human study 样本量小但合理，具体数值（85.6% vs 90.9% progress, 80% vs 80.6% success）已给出，但没有置信区间
- ⚠️ **"Emergent strategy"**：Fig. 13 举例 compelling（垂直 vs 倾斜夹持），但没有系统性地量化 "策略不同" 的程度
- ⚠️ **"Compositional task generalization"**：air fryer 案例定性 compelling，作者自承溯源困难；没有专门的组合泛化 benchmark
- ❌ **"Step-change in generalization"** / **"first signs of compositional generalization"**：标题与引言里的营销修辞，缺少可操作的度量方式

## Notes

- **最核心的 takeaway**：如果 "how" 可以被 prompt 表达，训练数据的 quality 就不是硬约束而是可控 condition。这把 imitation learning 的 "garbage in, garbage out" 重新框定为 "label the garbage properly"。是一条可能定义下一代 VLA 数据配比策略的 principle。
- **World model → visual CoT** 这条线值得单独深究：当语言与 policy 偏置冲突时，图像目标是 anchor。推演到自主 agent，这可能是 "VLA 的 reasoning chain" 的关键载体，类似 LLM 的 scratchpad 但在 pixel space。
- **Compositional benchmark 是 open problem**：作者的 "不必区分组合与 remixing" 是务实但哲学回避。社区需要类似 SCAN / COGS 的专用 composition benchmark，否则 "emergent composition" 永远停留在 anecdote。
- **Metadata 自动化**：mistake 与 quality 目前靠人标，未来可能由 VLM scorer 或反思式 LLM 自动标注——值得关注。
- **Open question**：π0.7 的 steerability 是否能被外部 LLM agent 直接利用？如果 high-level policy 本身是 LLM，那 coaching → 自主 的 pipeline 就直接接上 agentic 系统了。这可能是 VLA + LLM agent 的一条自然融合路径。
- 开源 checkpoint 发布后应重跑关键消融（特别是 no-metadata 那组）验证数字。等 arxiv 正式版发布后补充精确数值表。
