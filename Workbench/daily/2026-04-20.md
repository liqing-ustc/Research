### CoT Degrades Spatial
- **Title**: Chain-of-Thought Degrades Visual Spatial Reasoning Capabilities of Multimodal LLMs
- **Institutes**: IIT Hyderabad, Microsoft Research India
- **Source**: [link](https://arxiv.org/abs/2604.16060)  📄 arXiv   **📒 论文笔记**: [[2604-CoTDegradesSpatial]]
- **核心**: 17 个模型 × 13 个 spatial benchmark 的统一评测发现 CoT prompting 在 visual spatial 任务上平均掉 3%，8 个 RL-trained MRM 中 7 个打不过它们共同的 Qwen2.5-VL-7B backbone（62.68%）。真正的 contribution 是 **No-Image++ ablation**——把图替成全灰、并给"Cannot determine"选项作为 GT，结果 non-CoT backbone 76.41% 会乖乖选"看不见"，但所有 MRM 都塌到 5.55%–11.35%，继续编造坐标和 spatial relation。进一步在 GPT 家族上验证 GPT-5 / GPT-5-nano 也出现 CoT 掉点，并观察到 proprietary trace 比开源 MRM trace 短一个数量级（350 vs 3600 chars）。
- **锐评**: 问题选得锋利，No-Image++ 是近期少见的**干净 falsification test**——单独一个 shortcut ablation 可以被怼，但"backbone 76.41% vs MRM 5–11%"这种量级差距基本堵死了 confounder 的口子，且所有 MRM 共用 Qwen2.5-VL-7B backbone 控制得很好，3 seeds 的 std 都在 0.1–0.5% 之间，3% 的 gap 在噪声之外。但是锤子的精度只有 behavioral——**完全没碰 attention / feature level**，"CoT 抑制了 vision encoder output" 仍停在现象学；"冗长 trace → hallucination" 也只是 character-count correlation，该做的 length-intervention 实验（把开源 MRM 截到 350 chars 看掉点是否恢复）没做。另外 13 个 benchmark **全是 MCQ**，本身就在放大 text-shortcut——真正的 spatial reasoning 如 open-ended distance / coordinate prediction 根本没测，所以"MRM 没有 generalized spatial intelligence"的 claim 严格来说只对 MCQ-spatial 成立。Table 1 的"7/8 输给 backbone"里 R1-Onevision 掉 -15.8% 是个 outlier（它的 Modality-Bridging 本来就是 text-only distill，在 spatial 上输是 trivially expected），剔除后 6 个 MRM 的 Δ 都在 -2% 内，narrative 被稍微放大了。最大的遗憾：结论"need vision-centric reasoning"是纯 negative result，作者自己在 conclusion 提的 visual verifier / visual PRM 连个 pilot 都没做，停在 complaint paper 而非 reformulation paper。但对"用 R1 recipe 做 spatial reasoning"这个方向，No-Image++ 已经足够让人停下来重新想想——对我自己的 agenda 来说这是**一记必须吸收的 negative signal**。
- **评分**: 2 → 👀 重要

### Do VLMs Truly Reason
- **Title**: Do Vision-Language Models Truly Perform Vision Reasoning? A Rigorous Study of the Modality Gap
- **Institutes**: NTU; Alibaba-NTU ANGEL; Tongyi Lab, Alibaba
- **Source**: [link](https://arxiv.org/abs/2604.16256)  📄 arXiv   **📒 论文笔记**: [[2604-DoVLMsTrulyReason]]
- **核心**: 构造三模态严格等价（text-only / image-only / image+text）、人工校对 250 道的 cross-math puzzle benchmark（CrossMath），用 hop-stratified CoT 作为 gold process supervision，在 Qwen 家族 6 个变体上测得一致现象——image-only Macro Acc 比 text-only 低 60-80 pt，且 image+text 普遍 < text-only。三路论证（OCR 本身准、SFT 有救、准确率随 hop 数塌陷）把失败归到 reasoning 而非 perception；image-only SFT+GRPO（position-weighted reward）在 Qwen3.5-9B 上把 Macro 从 3.2 拉到 50.4，但仍比同模型 text-only 76.4 差一截。
- **锐评**: 三模态严格等价 + 人工校对 + hop-stratified 是真 rigorous 的部分，这点比 MathVerse/EMMA 那套 entangled benchmark 干净得多，"image+text < text-only" 的 headline finding 也足够反直觉。但 "rigorous" 三分之二是真的、三分之一是标题党——probe 的几个关键控制没做到位：(1) 所有 main table 只测 Qwen 一家，就敢下 "current VLMs" 的 universal claim，GPT-4o / Gemini / Claude / InternVL 一个都没有，外推性被自家 vision module 的 bias 绑死；(2) "image+text 掉点"归因成 "visual distractor 干扰内部逻辑" 纯属 narrative，没有 attention / attribution / context-length 的 causal probe，可能被 multimodal prompt prefix 或 context budget confound；(3) 任务本质是 color-coded symbolic grid，"vision-first" 是词，不是事实——去 border 直接掉 5 pt 已经亲手证明模型靠**显式视觉 delimiter** 做切分，搬到自然场景（chart、geometry diagram、物理 scene）什么都没验证；(4) 最尴尬的 smoking gun：image-only post-training 让 **text-only 也从 44 涨到 76.4**——如果真修复了 visual grounding，text 通路不该同步受益这么多，更可能是学到了 CoT 格式而非 visual reasoning，但论文对这个现象只字不提；(5) Hop 分布极度 skewed（1-hop 占 77%，4+hop 只 4%，尾部样本量~10），"accuracy 随 hop 衰减"曲线的极端段置信区间完全没给；(6) OOD 增益 +1.4 / +2.5 对一个 claim "揭示 modality gap 本质" 的工作来说偏小。相对之前 VISTA-Bench / PuzzleVQA / "Bring Reason to Vision" 等 modality gap 文献，新信息主要是**三模态严格等价的 benchmark 设计 + image+text < text-only 这个 finding 的定量化**，不是范式级的前进。中等价值：**适合当作 modality gap 问题的一个干净 data point 存档**，但它揭示的现象本身在社区已有 prior，别把 "rigorous" 理解成 "definitive"。
- **评分**: 2 → 👀 重要

### HY-World 2.0
- **Title**: HY-World 2.0: A Multi-Modal World Model for Reconstructing, Generating, and Simulating 3D Worlds
- **Institutes**: Tencent Hunyuan
- **Source**: [link](https://arxiv.org/abs/2604.14268)  🔥 HF Trending ⬆️ 95   **📒 论文笔记**: [[2604-HYWorld2]]
- **核心**: 开源的 offline 3D world model，四阶段 pipeline（HY-Pano 2.0 → WorldNav → WorldStereo 2.0 → WorldMirror 2.0 + 3DGS）把 text/single-image 变成 navigable 3DGS 场景。核心 insight 是 Keyframe-latent VDM——放弃 Video-VAE 的时空压缩改成稀疏 keyframe 独立 spatial VAE，快 camera motion 下保细节。单场景端到端 ~712s on H20，claim 对标闭源 Marble。WorldMirror 2.0 的 normalized RoPE 让 multi-res 推理从 extrapolation 变 interpolation，7-Scenes/NRGBD/DTU/normal 全面刷新 open-source SOTA。
- **锐评**: 跟 Marble/Genie 的本质差距就是——**这不是 world model，是 navigable scene generator**。没有 dynamics，没有 action-conditioning，场景生成完就冻结成静态 3DGS；title 里的 "Simulating" 对应的是 mesh collision detection + character locomotion，和 Genie 3 的 interactive event 完全不在一个层次，严重的 marketing overreach。"3DGS-based" 到底是不是真 world model？答案是**视频生成 + depth 对齐 + Gaussian 拼装**的工程整合——WorldStereo 2.0 生成 keyframes、WorldMirror 2.0 出 depth、RANSAC linear alignment 对齐到 panoramic point cloud、MaskGaussian 训 3DGS，每一步都能找到 open-source 前身（Uni3C / VGGT / FlashWorld / Gen3C）。"navigable" 的交互性止步于 camera control + collision-aware character walking 在单房间/单街角尺度，Fig 22 的演示 range 一眼看穿，没做长距离 exploration 或 cross-scene navigation。和 Marble 的对比**整个 Sec 8.1.5 零定量数据**，就两张 cherry-picked 定性图，既不做 user study 也不跑 WorldScore（作者自己引过），"comparable to Marble" 的 claim 纯靠嘴硬。WorldNav 的 5 种 heuristic trajectory 是典型"规则驱动刚好 work"——72 候选、45° pitch、120° FoV 全是硬编码，每个常数都没 ablation，真正 scalable 的 active view planning 应该是 learned policy。系统/工程 vs science 的比重大概 8:2——真 insight 就 Keyframe-VAE 一个（+Tab 7 的 PSNR 1.8dB 差距说服 spatial vs temporal SSM 拼接），其余是 competent 的工程优化。**但不能否认**：WorldMirror 2.0 刷了一片 reconstruction benchmark 是硬通货（high-res AUC@30 从 66.29 拉回 86.89，ScanNet normal error 12.3 击败 DSine），77% Gaussian 削减 -0.15 dB 的 MaskGaussian trade-off 也很漂亮，代码权重全开源对社区是实打实贡献。HY-World 1.0 + 1.5 + Uni3C + WorldMirror 1.0 的 incremental integration，但**整合得极好**——所以是"重要"而不是"必读"。
- **评分**: 2 → 👀 重要

### SpatialEvo
- **Title**: SpatialEvo: Self-Evolving Spatial Intelligence via Deterministic Geometric Environments
- **Institutes**: Zhejiang University, StepFun
- **Source**: [link](https://arxiv.org/abs/2604.14144)  🔥 HF Trending ⬆️ 62   **📒 论文笔记**: [[2604-SpatialEvo]]
- **核心**: 把 ScanNet 家族的 point cloud + camera pose 当 deterministic oracle，16 类 spatial task 每类写死一组几何验证规则，用程序化 GT 替掉 self-evolving 常见的 majority-vote pseudo-label；单个 Qwen2.5-VL 参数共享地扮 questioner / solver 两个角色，GRPO + task-adaptive scheduler 驱动 online 自进化。9 个 benchmark 平均分 3B/7B 拿到 51.1 / 54.7，VSI-Bench 7B 达 46.1；`w/o Physical Grounding` 消融让 VSI-Bench 从 46.1 崩到 18.8——把 thesis 钉死。
- **锐评**: "Self-evolving" 基本是营销皮——16 类 task + 验证规则全人工定义，questioner 的自由度只剩 entity phrasing，不是 AlphaZero 意义的自进化，更像是"programmatically-verified online RL"。真·核心是 DGE 这个 geometric oracle，self-play 架构（w/o Questioner/Solver 各只掉 1.6/3.2）在 ablation 面前完全被 physical grounding（↓5.1）碾压，这点作者自己的数据都承认。Delta vs SpatialVLM/SpatialRGPT/SpatialBot 清楚：前者是"离线批量合成 VQA"，SpatialEvo 是"在线动态生成+几何 verification"；但 Table 3 里用 DGE 离线生成数据做 SFT 拿到 43.9，online RL 才 46.3——只差 2.4 分，"online 必须 > offline"这个 narrative 其实站不太稳。几何环境的 coverage 更是死穴：只能在 ScanNet 级别高保真静态室内场景工作，outdoor / dynamic / sparse point cloud 直接废，作者 Appendix 承认但正文没划重点。16 类 task 是 methodology-shaped 选出来的"能被几何 operator 闭合定义"子集，affordance / functional / commonsense spatial 这些真硬的推理一个没碰，在这个 subset 上 work 不等于在 spatial reasoning 全集上 work。另外 invalid-question 的 explanation reward 又引回了 LLM judge，和开篇"摆脱 model consensus"的 framing 自相矛盾（虽然贡献只 0.4 点，直接砍掉更干净）。Parsing failure rate 整篇没给——"zero-noise" 的 framing 实际上 conditional on LLM entity parser 不出错，这层 hedge 没量化。3B VSI-Bench 被 SpatialLadder 反超 (39.2 vs 45.7) 正文没醒目交代，藏在 §5.1 task-coverage 的解释里。但工程实打实：4K 场景、3B/7B 权重、160K 数据、完整训练栈全开源，MMStar/RealWorldQA 基本不掉（对比 ViLaSR 在 V-STAR 78.5→35.6 的崩盘），Table 4 的 4-iter curriculum 防止 stagnation 证据扎实——作为 embodied self-improvement 的 primitive（把 simulator 当 verifiable reward source）有 reference value。
- **评分**: 2 → 👀 重要

### ClawGUI
- **Title**: ClawGUI: A Unified Framework for Training, Evaluating, and Deploying GUI Agents
- **Institutes**: Zhejiang University
- **Source**: [link](https://arxiv.org/abs/2604.11784)  🔥 HF Trending ⬆️ 138   **📒 论文笔记**: [[2604-ClawGUI]]
- **核心**: GUI agent 全栈 open-source harness——ClawGUI-RL（online RL 训练，支持 Docker emulator 和真机）+ ClawGUI-Eval（6 benchmark × 11+ 模型的 Infer/Judge/Metric 解耦评测，达 95.8% 复现率）+ ClawGUI-Agent（12+ 聊天平台部署 + 个性化 memory）。算法层唯一新东西是 GiGPO + PRM 的组合——two-level hierarchical advantage（episode-level + anchor-state-clustered step-level）叠上 Qwen3.5-72B zero-shot 当 process reward judge。ClawGUI-2B 在 MobileWorld GUI-Only 上拿到 17.1% SR，相对 MAI-UI-2B base 的 11.1% 涨了 6 个点，并比 UI-Venus-72B (16.4%) 略高。
- **锐评**: 典型的 infra-paper 伪装 research-paper——真 contribution 是 95.8% 复现率那张表，它把 "GUI eval 结果不可比" 从玄学降级成配置未披露的工程问题，这点诊断价值远高于 17.1% SR。但方法层几乎全是 borrow：GiGPO 是别人的、PRM 迁移是别人的、verl-agent 是别人的，自己做的主要是整合。硬伤清单：**(1) PRM 完全没验证**——所谓 dense step-level supervision 的效力建立在 Qwen3.5-72B zero-shot 能可靠判断 GUI action contribution 这个未测假设上，false positive rate、reward hacking 压力测试一个都没做；**(2) 核心 ablation 耦合**——GRPO→GiGPO 的 +2.6 SR 同时引入 hierarchical advantage 和 PRM step reward，没有 "GiGPO w/o PRM" 或 "GRPO + PRM" 的解耦，归因不可分；**(3) 真机训练是 claim 不是事实**——abstract 吹 "validated support for real physical devices"，但 ClawGUI-2B 的 flagship 实际上 100% 跑在 64 个虚拟 Android emulator 上，真机训练的 SR 数字和收敛行为一个字都没给；**(4) "Small well-trained beats large untrained" 的叙事偏移焦点**——拿 RL-post-trained 2B 打 pure-SFT 的 32B/72B 本来就不 apples-to-apples，Doubao-1.5-UI-TARS 26.3% 这条 end-to-end 对手作者刻意没讨论；**(5) "first open-source GUI agent RL infrastructure"** 在同期 MobileGUI-RL / ComputerRL 等被作者自己 cite 的情况下，这个 first 的 scope 定义太宽松；**(6) Personalized memory 完全没 evaluation**——vector embedding + top-k 检索 + 去重合并全是 description，没有召回率或长期交互下的 SR 提升数据。Strengths 也要说：spare server rotation / periodic teardown 这类细节写得出来说明真跑过长训练、Infer/Judge/Metric 三段解耦对后续研究者是实打实方便、11+ 模型预计算 prediction 全开源省掉复现时的 GPU 消耗。整体判断：作为 GUI agent 领域的 infra 工具 6–12 个月后看社区采纳度再定论，作为 research paper 的 methodological novelty 偏薄。
- **评分**: 1 → 💤 可参考

### SkillClaw
- **Title**: SkillClaw: Let Skills Evolve Collectively with Agentic Evolver
- **Institutes**: DreamX Team (AMAP / Alibaba)
- **Source**: [link](https://arxiv.org/abs/2604.08377)  🔥 HF Trending ⬆️ 279   **📒 论文笔记**: [[2604-SkillClaw]]
- **核心**: 在 OpenClaw（Claude Code stand-in）生态里把多用户 session trajectory 按 referenced skill 聚合成 natural-ablation 证据，由一个 agentic evolver 在共享 skill markdown 上选 {refine, create, skip}，夜间在真实 tool 环境做 A/B 对拍 validator，仅 Accept 入池推送给所有 agent；6 天 6 轮在 Qwen3-Max + WildClawBench 的 4 个类目上相对 Day 1 分别涨 11.7% / 52% / 88.4% / 33.3%。
- **锐评**: 硬伤一堆。(1) Abstract 吹 "continuous improvement over 6 rounds"，Table 3 里 4 类中 3 类第 1 或第 2 次提升后立刻 plateau 5 天——真实是 "一次性 rewrite + 后续全 reject"；(2) 那个头条 +88.41% 是分母 11.57% 放出来的，绝对 gain 只有 +10 个百分点，拿来做 HF card 宣传标题属于刻意误导，Creative Synthesis 整个提升还只来自一个 `validate-tmp-workspace-inputs` 的 workspace preflight skill；(3) validator 用的就是 daytime 用户跑过的任务子集——在 test traffic 上调 skill，没有任何 held-out 泛化测试，Accept 的 skill 完全可能 overfit 这批 procedural quirk；(4) 6 个类目只报 4 个，Productivity Flow / Code Intelligence "to be included in future version" 大概率是没涨或掉了；(5) 没有 baseline 对比——和 single-user Reflexion、和纯 static skill + RAG 比都没做，所以"collective"带来的增益 vs 简单多轮 refinement 完全不可分辨；(6) 只在 Qwen3-Max 单 backbone 跑，没换模型，agentic evolver 也没做 ablation；(7) N=8 用户根本撑不起 "collective evolution" 的立论，natural ablation framing 在这规模下是空谈；(8) Accept 率粉饰——Table 5 Day 3 / Table 7 Day 4 "candidate (none)" 也被标 Accept 凑数。真正亮眼的只有两点：natural-ablation 的 framing 和 real-execution validator（不是 LLM-judge）的工程诚实度；appendix 的 Agentic Evolve Prompt 反而是全文信息密度最高的 artifact，能看到作者自己观察到 evolver 会把正确 API 改成 "go read utils.py" 这种结构性翻车，被硬塞进 prompt constraint 里压着。跨用户 skill evolution 的两大坑论文只字未提：**privacy**（用户 Slack / 私有工具轨迹回传到 central evolver）和 **skill quality drift**（少数用户的环境 quirk 可能污染 global skill pool，best-skill 同步给所有人隐含了用户环境同构假设，在真 Claude Code 用户群里一推就炸）。与 Voyager / ExpeL / AgentKB / SkillWeaver / MetaClaw 等同期工作的实际 delta 就是 "per-user 改成 central + execution validator"——工程 patch 可以，但作为 ML paper 的方法贡献偏薄；命名 OpenClaw/CoPaw/IronClaw/...Claw 一长串明显在蹭 Claude 流量。
- **评分**: 1 → 💤 可参考

### PersonaVLM
- **Title**: PersonaVLM: Long-Term Personalized Multimodal LLMs
- **Institutes**: Nanjing University, ByteDance
- **Source**: [link](https://arxiv.org/abs/2604.13074)  📰 HF Daily ⬆️ 19   **📒 论文笔记**: [[2604-PersonaVLM]]
- **核心**: Qwen2.5-VL-7B 上套一层 agentic wrapper：四类 memory (core/semantic/episodic/procedural) + Big-Five 人格向量用 cosine-decay EMA 更新 + 多步 retrieval loop，SFT (78k) + GRPO (6k, LLM-as-judge reward) 两阶段训练。自建 Persona-MME benchmark，声称 128k 下比 Qwen baseline +22.4%、比 GPT-4o +5.2%；OOD 的 PERSONAMEM 上 +2.0% vs GPT-4o。
- **锐评**: 典型的"系统堆料 + 自建 benchmark"paper，三个致命问题：(1) **Evaluation 循环**——训练数据和 Persona-MME 都出自同一个 Seed1.6-thinking 合成 pipeline、都从 PersonaHub 抽 persona，+22.4% 那个大数字基本不可信；真正 OOD 的 PERSONAMEM 只 +2.0%、P-SOUPS 只 +2.46%，margin 缩水一个量级。(2) **四类 memory 是过度工程**——ablation 一摊开就穿帮：Core/Procedural/Semantic 各去掉只掉 1-2%，Episodic 扛了 12.4%。真正 work 的就是"按 topic 切片 + timestamp + keyword 的 RAG"，MemGPT-style 的 core/persona 分块和 procedural 基本是装饰。所谓 "long-term personalization" 根本不是长上下文记忆，是带时间戳的分段 RAG，和 memory-augmented LLM 线（A-Mem、MemGPT、Memory OS）的本质区别仅在于"补上了多模态 + 加了 personality 分支"。(3) **Big-Five 人格建模未被证伪**——把用户画像压成 5 维浮点向量 + EMA 听起来 clean，但论文没做任何 sanity check（同 persona 不同对话段的 $\mathbf{p}'$ 方差是多少？和心理学 ground truth 的一致性？），典型 "quantitative facade"。加上 RL reward 用 Qwen3-30B-A3B 打分、open-ended eval 用 Gemini-2.5-Pro 打分、PersonaVLM 强制 `<think>/<retrieve>/<answer>` 结构化输出——两个 LLM judge 都会偏爱这种格式，79% win rate vs GPT-4o 的数字要大打折扣。128k PERSONAMEM 上 Qwen baseline 只有 3.08 分（基本 random），所谓"巨大提升"有一半是在修 long-context collapse 而非做 personalization。未开源，repo 404。
- **评分**: 1 → 💤 可参考
